# 구현 계획 - 뉴스 번역 세이프가드 및 서버 복구

## 1. 문제 분석
- **현상**: 한국어 모드에서 뉴스 티커가 보이지 않음.
- **원인**: 
    - 백엔드의 `analyzer.py`에서 뉴스 번역(AI) 실패 시 빈 리스트(`[]`)를 반환함.
    - 프론트엔드 `NewsTicker.jsx`에서 뉴스 데이터가 없으면 컴포넌트 전체를 렌더링하지 않음.
    - 또한 프론트엔드 서버(Vite)가 중지되어 있었음.

## 2. 해결 방안
- **백엔드 수정**: 뉴스 번역 시 예외가 발생하거나 결과가 부실할 경우, 빈 리스트 대신 원본 영어 뉴스(`raw_news`)를 반환하도록 수정.
- **서버 복구**: 중지된 프론트엔드(5173) 및 백엔드(8001) 서버 재시작.

## 3. 상세 변경 사항
- `backend/analyzer.py`: `fetch_crypto_news` 내의 `try-except` 블록 및 필터 로직 보강.
- `frontend`: `npm run dev` 실행.

## 4. 결과 확인
- `curl` 요청을 통해 한국어 모드에서도 뉴스 데이터가 정상 반환됨을 확인.
- 프론트엔드 접속 시 뉴스 티커가 노출됨을 확인.
